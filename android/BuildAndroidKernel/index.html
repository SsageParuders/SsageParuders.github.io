# 安卓内核介绍

  • 真要详细且专业的讲讲安卓内核 & Linux 内核
    鄙人自愧 才疏学浅 不敢献丑
    但要是说普及一些基础概念 还是想唠上两句

  • 安卓是 Linux 的嵌入式发展
    也就是说 安卓系统中包含了一个 Linux 的内核 (Kernel)
    平时我们使用的 蓝牙 Wi-Fi 等等功能
    无不依赖于 Kernel 中驱动提供的支持

    举个栗子

    小米官方提供的 Kernel 开源库
    其中由于未知原因 缺失 wlan 模块
    【issues 传送门】
    我在测试学习过程中
    编译了一个 sagit-p-oss 内核
    (不同的机型对应不同的内核)
    (该内核专门对应小米 6)
    并且刷入了我的小米 6
    结果由于内核模块缺失
    Wi-Fi 驱动无法加载
    我的小米 6 丧失 Wi-Fi 功能

  • 也就是说 安卓内核中提供的诸多模块驱动
    对整个安卓系统有较为重要的影响
    那么说:

    • 如何编译安卓内核 (Android Kernel)
    • 如何编译内核驱动模块并且刷入

    这里我根据自己摸索的经验结合 GitHub 上的 issues

    作出几点总结:
    1. 操作系统是否正确
      前文吐槽中提到 操作系统不同
      是会导致编译失败的
      这里建议 Ubuntu 18.04.6 LTS
    2. 如果是头文件错误
      根据 issues 表示
      应该在 make 的时候指定 O= 参数
      (我这里的命令已经指定了该参数)
      或者将头文件的 <> 改为 ""
      详情请见 【issues 传送门】
    3. 如果是内核源码的语法错误
      理论上来说 能被开源的代码
      基本上是开发者已经编译成功过的代码
      排除极少数可能 应该语法上的错误不会存在
      如果报出语法错误 很有可能是
      你使用的编译器不正确
      这种情况在我们团队合作的时候出现过
      不同的内核源码 应该视情况选择对应的编译器
      编译器的选择标准 上文已经介绍 这里不再赘述
    4. 极少数情况是源码真的错了
      这种情况建议看看 issues 或者向官方开发者联系

# 如何刷入内核模块

刷入内核模块的思路大概分为两种

  • 刷入自编译的内核与模块
    自己编译的内核 你自然有他的 key
    有了 key 自然可以对模块进行正确的签名
    甚至你还可以在编译时 选择对应参数
    不开启内核校验 这样任何模块均可刷入
  • 绕过内核签名校验和 crc 校验
    利用 Kernel 的漏洞 绕过内核的校验
    实现该方案的途径并不唯一 有很多漏洞 / 骚操作可以利用
    但是无论如何 使用该方案需要更高的技术要求

下面 我来分别简介两种方案

# 替换自编译的安卓内核

如果你选择了替换内核的方式刷入内核模块
那么 你有两条方式可以选择

  1. 对模块签名
    即使是各项环境相同 每次编译内核产生的 key 都是不一样的
    使用 key 对模块进行签名 如此也保证了内核安全
    使用自己编译的内核产出的 key 对后续编译的动态模块进行签名
    这样 签名后的模块自然可以刷入到自己编译的对应内核中去
    由于博主没有在小米内核中找到对应的签名 key
    可能是我眼浊 没看到 但是我严重怀疑小米的 key 重命名了
    因此 这里不对这种方式多做说明 (未来有时间可能补上)

  2. 关闭校验
    将自己编译的内核关闭校验
    在对内核源码进行 make 时
    一般都会指定配置文件
    比如小米 6 在 make 的时候 配置文件就是 sagit_user_defconfig
    配置文件中有各项编译参数
    其中有如下三个参数是有关于内核签名的

    模块签名验证功能 配置为 "y" 后 才有后面的子选项可以配置

    是否验证模块的签名
    配置为 "n" 时 对于未签名或者使用无效的 key 签名的模块 内核都允许加载
    但是 kernel 会被标记为被感染 (tainted) 并且使用商业许可的模块也会被标记为被感染
    若配置为‘y’ 则只有正确签名的模块才能被允许加载
    其他的模块将被拒绝加载 并且 kernel 会打出错误信息

    若配置为 "n" 则在 build kernel 的过程中
    所有的模块都不会被签名 需要自己手动签名
    若被配置为 "y" 则在 build kernel 的 "modules_install" 阶段
    所有的模块都会被自动签名

    如上所述 只需要在 make 前 将配置文件中的
    CONFIG_MODULE_SIG_FORCE 参数改为 “n”
    制作出来的内核即可刷入任意模块 不过会被标记
    因此 较为适合开发环境 不适合发行环境
    对我们开发者来说 这项参数为 "n" 即可

  3. 一些补充

    1. 注意，Makefile 中要加上 CFLAGS_MODULE=-fno-pic 选项，不然编译好的 模块 relocation 节会错误
