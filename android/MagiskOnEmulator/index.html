Android studio 原生模拟器 (Emulator) 刷入 Magisk 获取 root

首页 Android

最近在研究安卓方面的逆向
在练习和研究阶段需要设备拥有 root 环境
固然是可以真机刷入 root 环境，电脑连接真机
但是虚拟设备 (Emulator) 刷入 root 环境显然更加便于研究
于是开始折腾 Android studio 原生的模拟器如何刷入 root 环境

# 创建不具有 Google play 的设备

在查阅 Google 官方的 Android 开发者文档以后
找到了如下资料

也就是说 Google 官方规定：带有 Google play 的镜像是安全的，不可提权获取 root 的，如果在开发过程中想要获取 root 环境，可以创建不带有 Google play 标识的虚拟设备，这种设备可以获取 root 权限

  • 博主已经亲测尝试 不带有 Google play 的设备 可以直接获取 su 身份

# 安装 SuperSU 获取 root 权限

在查阅相关资料的时候
发现国内大部分的文章资料都是采用刷入 SuperSU 的方式强行获取 root 权限
但是采用这样的方式 会有如下缺陷

  • SuperSU 貌似停止更新 因此 SuperSU 获取 root 的方式只停留在安卓 9.0 以下
  • 貌似依然无法突破 Google play 的限制 且 abi 只能创建 x86 构架
  • 由于博主不想尝试这种过时的方法 因此暂未实践 这里也不再赘述 想要查阅相关资料的可以自行搜索

# Emulator 刷入 Magisk 获取 root

提到物理机如何获取 root 就不可避免的想到大名鼎鼎的 magisk 项目
那么 magisk 能不能在 Android studio 原生模拟器运行呢？
获取 magisk 甚至可以刷入 xposed 框架？
我查了一下 magisk 的 Git 上面的 issues
发现真的有一些相关资料
并且找到了一份相关 git 项目 ——MagiskOnEmulator
按照 README 操作 成功在 Android studio 带有 Google api 的虚拟机上获取 root
下面 我将对相关操作进行讲解 (相关工具会放在文章底部)

# 提取 boot.img

在你的 Android sdk 路径下提取对应的 boot.img 镜像文件

\system-images (系统镜像目录)\android-30 (对应你的镜像版本)\google_apis\x86 (到达镜像目录)

复制 boot.img 到 MagiskOnEmulator 项目目录下
