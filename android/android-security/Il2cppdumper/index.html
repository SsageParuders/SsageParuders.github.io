Il2cppdumper 使用教程

本文所指的 Unity 安全攻防 暂时只包括移动端 (尤指 Android) 不包括 PC 端攻防逆向

面向 Unity 开发的游戏做逆向安全
难免需要对游戏的 native 层 做一下 dump
老版本的 Unity 游戏开发 都是典型的依托于 mono 编译
后受到版权 / 效率 / 维护等各方面因素影响
Unity 统一采用 IL2CPP 体系 将中间语言 IL 转化为 cpp 执行
因此 对目前的 Unity 系 进行 dump 时 主要目标是 il2cpp.so 文件

本文仅且只做技术安全研讨 愿每一位安全从业者静有所思
在本文中涉及到的 Unity 游戏 理应对其包名等特征信息作脱敏处理
如相关负责人对本文持有侵权怀疑 请联系本人进行内容整改

# 项目介绍

Unity 逆向中最常用的工具应该是 Perfare 大佬的项目 Il2cppdumper
和对应的 Android 版项目 Riru-Il2CppDumper
本资料偏教程向 主要归类总结 Il2cppdumper 的各种使用方式
并浅谈 Unity 项目安全的攻防
Il2cppdumper 博主 fork 后地址: 点此链接
Riru-Il2CppDumper 博主 fork 后地址: 点此链接

# 基础知识

将 Android 端的游戏后缀 apk 修改为 zip
解压以后即可看到游戏内部目录结构

  • 在游戏的 assets/bin/Date 目录下
    存在很多没有命名规律的未知文件
    将他们以记事本形式打开 如图所示

# 如何提取 Unity 项目的 global-metadata.dat 和 il2cpp.so

  • 在游戏的 assets/bin/Date/Managed/Metadata 目录下
    存在 global-metadata.dat 文件
    部分游戏可能对此文件进行加密 这个后面会提到
    只需知道这个目录下的文件就是 global-metadata.dat 即可

  • 在游戏的 lib/arm64-v8a (对应处理器构架) 目录下
    存在一个游戏的全部 native 层 so 文件
    我们要的 il2cpp.so 也在这个目录下

    已经存在了 dump 后的输出文件
    在 dump.cs 中 内部方法名和偏移已经一清二楚
  6. 除了 dump.cs ，还有一些其他文件 各有其他用处
    比如 il2cpp.h 是包含了结构体的头文件

    具体可以参考 Il2cppdumper 的 README

特别说明

在老版本的 Il2cppdumper 中 需要分模式运行
手工模式中 需要手动指定两处指针地址
新版的 Il2cppdumper 中 做了一定优化
不再需要手工模式 优化为全自动处理
但在 Unity 项目做了一定加密的情况下
手动利用反汇编工具寻找 Unity 源码特征
观察 Unity 特征规律能起到一定的解密作用

dump 的产物最多只能显示:
属性，方法类和接口以及他们在 so 中的偏移位置
但是无法显示内部具体逻辑
不过根据方法类 / 接口的名字
大致可以对其作用进行猜测 有助于逆向分析
想要了解内部实现的具体逻辑
应该用 dump 产物的信息去结合 ida 进行分析
相关分析手法后面有机会再详细记录

# 稍作加密的项目如何处理

安全保护可能分为如下几种

  • global-metadata.dat 被加密
  • so 文件被加固

下面我们来分类讨论

# global-metadata.dat 被加密的情况

在 il2cppdumper 中 如果出现 global-metadata.dat 被加密的情况
则会曝出如下错误:

  • 面对 global-metadata.dat 被加密的情况 大概有两种思路

    1. 找到 global-metadata.dat 的加密规律 并进行解密
      在 il2cppdumper 作者的博客中 有提到类似的情况:
    2. 尝试 Riru-Il2cppdumper 这个我们后面再说

  • 由于加密和解密的攻防并不绝对
    对于同一种文件 不同的人也有不同的加密方式
    所以 global-metadata.dat 的解密也没有什么通解
    只能说是每个人根据自己对 Unity 项目的源码理解
    反编译资源文件后对特征进行总结观察
    得到加密规律后进行解密

因此遇到 global-metadata.bat 加密的情况 建议尝试 Riru-Il2cppdumper
或另寻高人解密

# Riru-Il2cppdumper

一般情况下 只有对资源文件做了严格加密的 Unity 游戏
！！(比如 so 加固了 metadata 也加密了 一般 so 还可以转成 bin 解决 主要是 metadata 的加密难以处理)！！
才会用到 Riru-Il2CppDumper 进行 dump
因为根据 Perfare 的表述 该项目可以绕过几乎所有保护
遗憾的是 Riru-Il2CppDumper 进行的解密暂时还不算十分完善
你会发现 riru-il2cppdumper 出来的 dump.cs 的内存会比较小

缺陷(此时项目版本@90a8fb3)

不过对于常规的安全攻防是足够使用的了

    将版本区间前的值 覆写到 UnityVersion 上去
  4. 最终如下图所示:

  5. 点击 小锤子 进行 build 等待 build 结束
  6. build 结束以后 进入 out 目录下 zip 文件即为 Magisk+riru 模块

# 特别提醒

这款游戏的加密安全做的比较到位
我是受一位朋友委托 对这款游戏进行安全测试
在测试过程中 由于技术水平有限 被迫使用 Riru-Il2cppdumper
编译完成模块以后 在我的开发测试机中
对这款游戏的 dump 一次成功 直接输出了 dump.cs
但是朋友使用我的模块无法输出 dump.cs
起初以为是朋友的某种操作步骤出错
但是在写本篇博文的时候 对安全测试环境进行场景复现
发现自己同样也没有输出 dump.cs
因此稍作分析 发现:
由于开发测试机环境复杂 存在很多 debug 工具
该 Unity 项目对运行环境存在较为严格的检测
在检测到 debug 环境以后则会拒绝加载 il2cpp.so 并且闪退
导致无法输出 dump.cs
最后 在规避运行环境检测 正常加载项目的基础上
成功再一次输出 dump.cs

所以

注意事项

Riru-Il2cppdumper 要求 Unity 项目可以正常的完全加载
要求 il2cpp.so 完全运行
因此 攻击者要给目标项目制造一个安全的运行环境
或规避目标项目的一些运行环境检测

至此 Riru-Il2cppdumper 教程结束

# 浅谈攻防安全

攻防无绝对 技术无黑白
愿每一位安全从业者静有所思

# 对于安全防御方来说

对 global-metadata.dat 做必要的加密是有必要的
建议同时对 il2cpp.so 做加壳防御
将游戏内关键的一些方法名
另取名为怪僻的方法名 或对方法名做混淆
以此干扰逆向难度

# 对安全攻击方来说

想要流畅的应对各种 Unity 项目的防御手段
一方面需要掌握基础的 Unity 项目正向开发基础
另一方面需要去苦读 Unity 引擎源码
去了解 Unity 项目的特征
从特征出发往往能锻炼自主逆向的能力
