本文所指的 Unity 安全攻防 暂时只包括移动端 (尤指 Android) 不包括 PC 端攻防逆向

面向 Unity 开发的游戏做逆向安全
难免需要对游戏的 native 层 做一下 dump
老版本的 Unity 游戏开发 都是典型的依托于 mono 编译
后受到版权 / 效率 / 维护等各方面因素影响
Unity 统一采用 IL2CPP 体系 将中间语言 IL 转化为 cpp 执行
因此 对目前的 Unity 系 进行 dump 时 主要目标是 li2cpp.so 文件

本文仅且只做技术安全研讨 愿每一位安全从业者静有所思
在本文中涉及到的 Unity 游戏 理应对其包名等特征信息作脱敏处理

Unity 逆向中最常用的工具应该是 Perfare 大佬的项目 Il2cppdumper
和对应的 Android 版项目 Riru-Il2CppDumper

并浅谈 Unity 项目安全的攻防

# 如何提取 Unity 项目的 global-metadata.dat 和 li2cpp.so

  • 在游戏的 assets/bin/Date/Managed/Metadata 目录下
    存在 global-metadata.dat 文件
    部分游戏可能对此文件进行加密 这个后面会提到
    只需知道这个目录下的文件就是 global-metadata.dat 即可

  • 在游戏的 lib/arm64-v8a (对应处理器构架) 目录下
    存在一个游戏的全部 native 层 so 文件
    我们要的 li2cpp.so 也在这个目录下

# 普通的未加密项目

  1. 在 cmd 中打开到 Il2cppdumper 项目路径
  2. 创建 RES 目录用来存放 global-metadata.dat 和 li2cpp.so
  3. 创建 OUT 目录用来存放 Il2cppdumper 输出内容
  4. 在 cmd 中输入如图所示:
  5. 按照指令运行以后 可以看到我们的 OUT 目录下
    已经存在了 dump 后的输出文件
    在 dump.cs 中 内部方法名和偏移已经一清二楚
  6. 除了 dump.cs ，还有一些其他文件 各有其他用处
    比如 il2cpp.h 是包含了结构体的头文件
    DummyDll 中的 dll 经过反编译以后可以查看到更完善的方法类或接口信息
    具体可以参考 Il2cppdumper 的 README

特别说明

在老版本的 Il2cppdumper 中 需要分模式运行
手工模式中 需要手动指定两处指针地址
新版的 Il2cppdumper 中 做了一定优化
不再需要手工模式 优化为全自动处理
但在在 Unity 项目做了一定加密的情况下
手动利用反汇编工具寻找 Unity 源码特征能起到一定的解密作用

dump 的产物最多只能显示:
属性，方法类和接口以及他们在 so 中的偏移位置
但是无法显示内部具体逻辑
不过根据方法类 / 接口的名字
大致可以对其作用进行猜测 有助于逆向分析
想要了解内部实现的具体逻辑
应该用 dump 产物的信息去结合 ida 进行分析
相关分析手法后面有机会再详细记录

# Riru-Il2cppdumper

一般情况下 只有对资源文件做了严格加密的 Unity 游戏
才会用到 Riru-Il2CppDumper 进行 dump
因为根据 Perfare 的表述 该项目可以绕过几乎所有保护
遗憾的是 Riru-Il2CppDumper 进行的解密暂时还不算十分完善
