C/C++ 源码文件想要编译成可执行文件或依赖库 (dll,so...)
最原始的操作 自然是需要在命令行下
使用编译器 一条条命令的输入
如果每一份源码都使用命令行单独编译 效率难免低下
为了提高项目编译时期的效率 出现了很多的项目构建体系
比如：安卓开发时常见的 gradle 构建体系
再比如本次介绍的构建 C/C++ 项目使用的 CMake 体系

C/C++ 构建体系其实有很多种 这里只介绍 CMake
只是因为 CMake 体系比较常见 并且 CMake 体系跨平台可用
其他常见的 C/C++ 构建体系 比如微软的 MSBuild

下载和安装

下载和安装直接去官网即可 地址如下:
https://cmake.org
不同的平台下载不同的版本
PS: 我最喜欢 CMake 的交叉编译了～～

# CMake 体系介绍

大多数情况下 都是在命令行中使用 CMake
因此 CMake 属于一个命令行工具 类似于 ls mkdir
我们可以通过 man 命令来查看 CMake 的具体使用

在 Linux 命令后面跟上符号 >> + 自定义文件
即：将该命令的输出信息 写入到 自定义文件中

  • 在 CMake 体系中 编译流程大概依靠于两个重点
    1. 编译时 传入 CMake 的参数
    2. 编译前 针对项目所写的 CMakeList.txt
